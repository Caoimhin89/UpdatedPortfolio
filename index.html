MY PROJECTS

Learning new skills is not easy. To accelerate my progress and internalize concepts that I read about, I like to dive in and get coding. Making use of design patterns, best practices, and other coding techniques by using them right away in real projects is the most effective way I have found to learn them. For that reason, I have a number of independent projects that I have built, and others that are presently in development. Not all of my projects are public, but below you can learn a little bit about the ones that are.

  • An interactive map of the city of Ufa, where I studied for one year (2012-13). The application was created with the Knockout.js MVVM framework and Google Maps. The city's points of interest are retrieved dynamically from the Foursquare API and compiled into a list which may be filtered by the user based on a number of criteria or by a text query. Filtering the list automatically updates the markers on the map in accordance with the user's preferences. Clicking on a point of interest, either from the list or on the corresponding map marker, opens an info-window with content downloaded from Foursquare.

  • Currently in development. This web application is intended to aggregate historical articles, podcasts, and other media related to Classical Civilization - Greece, Rome, Byzantium, Persia, etc. It is built in Angular 2 with a Node backend and a Mongodb database. Users are able to log in, set their preferences for what content they are interested in, are given access to a discussion board, and may access their subscriptions in the "News Stand" section.

  • A simple clone of Space Invaders written in C# with the Unity Game Engine. Players face off against a squadron of enemy spacecraft which attack the player's ship in an attack formation with rapid-fire lasers. The player fires back with his own laser while trying to evade the enemy's attacks. The object of the game is to destroy as many enemy spacecraft as possible before the player's craft is, itself, destroyed. The game keeps track of the player's score and displays it on-screen.

  • A slightly modified clone of the classic arcade game, 'Frogger,' written in JavaScript and making use of the HTML5 Canvas API. The object of the game is for the player to escort the damsel-in-distress across the road without getting run-over by oncoming beetles. The player should also take care not to get distracted by enticing decoy game objects. The player's score is recorded and displayed to the user at the end of each round - which occurs every time the player successfully rescues the damsel-in-destress or he gets run over in the process.

  • A personal banking CLI application built in Java. The user can open an account, and perform a number of transactions. These include deposits, withdraws, and transfers. The banking data can also be transferred to or retrieved from a document using I/O, to ensure easy transfer of client information to another bank.

  • Currently in development. A shopping cart application that allows users to create a shopping list, browse and search products, all filterable by category, add the products to a shopping cart and go through a checkout process. Users can also leave a rating of those products which he or she purchased in the past. The front-end of this application is built with Angular 2, while the back-end utilizes Node and Express and connects to a Mongodb database.
